Snake Engine

Game Engine

SNAKE_ENGINE is a lightweight C++ game engine that I personally designed and implemented. It integrates 2D rendering, input, sound, text, collision, and state management into a single framework based on OpenGL 4.6. With a modular (dependency/tag–based) structure, projects can selectively use only the systems they need. It also supports advanced optimizations such as runtime glyph baking, batching & instancing, and frustum culling/spatial hashing.

While working at DigiPen Academy, I realized that the C++ framework provided to students was only distributed as a .lib file. This meant that students could not optimize performance or extend functionality, since the source code was inaccessible. To address this limitation, I decided to build my own engine from the ground up — one where the entire source is open, debuggable, and modifiable at any time.

The engine was built with a focus on learnability and debugging accessibility, designed around exposed headers and a resource registry system. Through sample projects, users can immediately test rendering pipelines, collision systems, and text rendering. It can also be compiled as a static library (.lib) and linked to external projects.

Key Features
  • State Transition Framework: Manageable at Load/Init/Update/Draw/Free/Unload level
  • Rendering Optimization: Batching identical Mesh/Material pairs, instancing
  • Camera & Frustum Culling: Camera2D–based off–screen object culling
  • Text Engine: FreeType runtime glyph baking (alignment & multi-line support)
  • Collision System: AABB/Circle colliders + Spatial Hash Grid + group/mask filtering
  • Sound: miniaudio–based playback/pause with tag–level controls
  • Asynchronous Loading: Background resource loading with progress tracking
  • Debug Draw & Logger: Collision visualization and unified log levels
Architecture Overview
  • WindowManager — GLFW/GLAD initialization, events, swap/clear
  • InputManager — Key/mouse input, scroll, world coordinate conversion
  • SoundManager — Sound loading, playback, and instance management
  • RenderManager — Resource registry, batching/instancing, debug drawing
  • StateManager — GameState switching, updating, and drawing
  • ObjectManager — Object lifecycle, collision, and render submission

The engine is still under continuous development, and has been validated through sample projects that demonstrate rendering, collision, text, and sound features within actual game scenes.
